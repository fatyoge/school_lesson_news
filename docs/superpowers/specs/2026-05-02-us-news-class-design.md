# 家长进课堂："看世界、想问题" 设计稿

> **项目代号**：school_lesson_news
> **文档版本**：v1.0
> **创建日期**：2026-05-02
> **听众**：小学四年级（按五年级认知设计）
> **场景**：学校"家长进课堂" · 一节课 40 分钟
> **基调**：中立启蒙——看世界、想问题，不下价值判断

---

## 1. 项目背景与目标

### 1.1 来源
家长应学校之邀，在自己孩子所在班级（四年级）开一节"家长进课堂"。家长本人选题：以美国一些"奇葩"新闻为切入点，引导孩子观察社会、思考公共问题。可能开多期，需要可重用的素材库。

### 1.2 核心目标
- **认知打开**：让孩子知道"原来世界上有这样的事"，破除对其他社会的扁平想象
- **思考启蒙**：从荒诞反差出发，引导孩子提出"为什么会这样""能不能改"这类问题
- **数字感建立**：通过具体对比让"$128B、16 年"这种抽象数字变得可感

### 1.3 非目标（明确不做的事）
- ❌ 不下"美国不好、中国好"的价值结论
- ❌ 不灌输任何政治立场或意识形态
- ❌ 不展开复杂的历史/制度背景（孩子无概念）
- ❌ 不讲不可证实的网络段子和都市传说
- ❌ 不渲染暴力/死亡/恐怖细节

### 1.4 关于"人权角度"的转化
原 idea 文档提及"从人权角度分析美国社会结构问题"。**"人权"作为抽象概念对四年级孩子无认知支点**，且在国内学校课堂直接使用风险高。本设计将其转化为孩子可感的具体维度：
- "公平"（消防费故事 / 救护车账单）
- "帮助 / 照顾"（基建公共服务故事）
- "规则"（前院种菜 / 接雨水故事）
- "安全"（校园演练 / Key Bridge 故事）

讲述中**完全不出现"人权"这个词**，但其精神内核（每个普通人是否得到合理对待）通过故事和数字自然传达。

## 2. 听众与约束

### 2.1 听众画像
- 年级：小学四年级，按五年级认知能力设计内容
- 数量：一个班 30-50 名孩子（随机座次）
- 注意力周期：~10 分钟一波
- 抽象能力：可处理简单类比、初步因果推理
- 数字感：弱——10 亿、16 年、200 公里这类数字需要具体参照系

### 2.2 时长约束
- 总时长：40 分钟（标准小学一节课）
- 单故事时长：3-5 分钟
- 互动总量：5-7 分钟（嵌入各故事的"想一想"环节）
- 缓冲：~7 分钟机动

### 2.3 政治敏感性约束
- 选题在国内学校属敏感话题，需家长事前与班主任沟通选题
- 全程避免使用：民主、独裁、资本主义、社会主义、自由、专制等抽象政治术语
- 避免直接对比"哪国制度好"，仅做数字对比

## 3. 产品定位

### 3.1 产物形态
**单一可移植 HTML 演示文件**，本地双击 `index.html` 即可全屏放映。

### 3.2 信息架构
**目录页 + N 个独立故事模块**——非线性可跳转结构：

```
index.html
├── 封面页
├── 目录页（卡片网格，可数字键直跳）
├── 故事 1（Section）
│   ├── 封面钩子页
│   ├── 故事铺垫页
│   ├── 反差揭示页（美国数据）
│   ├── 对比锚页（中美数据并排）
│   ├── 想一想页（互动）
│   └── 背景小知识页（可选）
├── 故事 2 …
├── …
├── 故事 9
└── 结束/总结页
```

### 3.3 关键特性
| 特性 | 说明 |
|---|---|
| **模块化** | 每个故事独立自洽，可单独跳转、单独跳过 |
| **可扩展** | 新故事 = 新增一个 Section + JSON 数据条目 |
| **讲师挑选** | 一节课讲 6-7 个故事，多期可换不同组合 |
| **数字键跳转** | 1-9 键直接跳到对应故事 |
| **离线可放映** | 图片本地化，上课不依赖网络 |
| **不要讲师备注模式** | 不使用 reveal.js speaker notes，所有讲述要点直接呈现在 slide 上 |

### 3.4 讲师工作流
1. 备课：打开 HTML 浏览所有故事，挑选本期讲哪 6-7 个，并写下顺序
2. 现场：从目录页按数字键进入故事 → 顺着 slide 讲 → 翻完跳回目录或下一个
3. 节奏控制：每个故事 3-5 分钟，最后一页"背景小知识"可跳过以省时

## 4. 单故事模板（标准化结构）

每个故事固定 5-6 张 slide：

| # | 页名 | 内容要素 | 时长 | 是否可跳过 |
|---|---|---|---|---|
| 1 | **封面钩子** | 标题（≤12 字） + 主图 + 1 个引发好奇的问句 | ~30s | 不可 |
| 2 | **故事铺垫** | 发生了什么——图 + 关键文字（≤30 字） | ~1min | 不可 |
| 3 | **反差揭示** | 美国的数字/事实（最有冲击力的那个数字） | ~45s | 不可 |
| 4 | **对比锚** | 中美数据并排——左红右白两栏 | ~45s | 不可 |
| 5 | **想一想** | 抛 1-2 个问题给孩子（A 举手投票 / B 开放提问，讲师二选一） | ~1-1.5min | 不可 |
| 6 | **背景小知识** | 制度/数字/收尾——后来怎么样了 | ~30s-1min | 可跳过 |

**走完全部 6 页 ≈ 4-5 分钟，跳过第 6 页 ≈ 3 分钟**。讲师靠"是否翻第 6 页"来弹性控制时长。

### 4.1 文字密度原则
- 标题 ≤ 12 字，一句话钩住人
- 正文每页 ≤ 30 字，靠图和讲师讲述补足细节
- 不写段落，多用数字 / 箭头 / 对比 / 单字爆点

### 4.2 互动设计原则
- **形式仅限两种**：
  - **A · 举手三选一**（最易管理）：例 "消防员该不该救？A 该 / B 不该 / C 一半一半"
  - **B · 开放提问**（讲师酌情）：例 "你觉得为什么会有这种规定？"
- **不做小组讨论**——四年级转头窃窃私语 1 分钟全班失控
- 单次互动 30-60 秒，全堂总计 5-7 分钟

### 4.3 收尾原则
每个故事尽量给"后来怎么样了"——传达"问题可被讨论、被改变"的信号，避免孩子产生世界恐慌。

## 5. 故事池清单

故事池起步 9 个（A 级 4 + B 级 5）。每个故事包含：标题、所属主题、美国数据、中国对比锚、互动建议、风险等级、来源。

### 5.0 风险等级定义

| 等级 | 含义 | 处理 |
|---|---|---|
| **低** | 内容轻松无刺激，适合任何氛围 | 直接讲，无特殊处理 |
| **中** | 涉及损失/账单/事故，有反差冲击 | 必须做"淡化处理"（详见 Section 8.2）；讲完接对比锚和"后来怎么样" |
| **中-高** | 涉及孩子心理可能受冲击的元素 | 严格按 filter_notes 走；讲完不留情绪沉重，立即接轻松故事或对比锚 |
| **高** | 涉及死亡/暴力/政治高敏感 | **不进故事池**（如校园枪击事件、Flint 水危机等） |

**当前故事池中没有"高"风险条目**——所有"高"风险题材均已剔除（详见 Section 5 末"剔除清单"）。

| 互动 B | "美国孩子为什么要练这个？"（开放提问，**讲师不展开枪击案细节**） | — |
| 严格过滤 | **聚焦"练什么"**（关灯、躲桌、装死）；**不展开"为什么要练"**（不讲具体枪击案、不展示伤亡）；不出现具体枪击事件名称、不出现遇难者照片 | — |
| 注意 | 这是整堂课最敏感的故事，建议讲完后立刻接相对轻松的下一个故事/收尾，避免孩子情绪沉重 | — |

### 5.10 剔除清单（明确不进故事池的题材）

| 题材 | 剔除原因 |
|---|---|
| 校园枪击事件本身（如 Sandy Hook, Uvalde） | 涉及儿童/青少年死亡的具体描述，对四年级心理冲击太大 |
| 胰岛素涨价致死案（Alec Smith 等） | 涉及青年死亡，题材沉重 |
| Flint 水危机 | 涉及种族 / 贫困议题，复杂度太高 |
| 东帕勒斯坦化学品泄漏（2023） | 涉及儿童健康受损 |
| "每年 X 万人失踪"传闻 | FBI NCIC 数据常被误读夸大（绝大多数当年找回），易传谣 |
| "猫做鬼脸违法"等奇葩法律段子 | 多为虚构或早已废止，不可信 |
| 大麻 / 毒品 / 性 / 酒精滥用 | 不适合小学四年级 |
| 选举舞弊 / 党争 / 罢免等政治议题 | 强政治立场，国内学校敏感 |

## 6. 视觉与技术栈

### 6.1 技术框架
- **reveal.js**（最新稳定版）—— 经典开源 HTML 演示框架
- 使用 CDN 加载或本地 vendor 一份（推荐本地化以确保离线可用）
- 不使用 build 工具（直接 HTML+CSS+JS，便于讲师自行调整）

### 6.2 视觉风格：新闻报道风
- 主背景：**深黑 / 深灰**（#0a0a0a / #1a1a1a）
- 主文字：**白色**
- 强调色：**红色**（#e63946 或 #d32f2f）—— 用于关键数字、美国端数据、警示元素
- 中国端数据：白色或淡绿（与红色形成对照）
- 字体：
  - 标题：思源黑体 / PingFang SC 粗体
  - 正文：思源黑体 / PingFang SC Regular
  - 数字：使用更大字号（强调感）+ 等宽字体（数字感）
- 图片：真实新闻截图、纪实照片，不加滤镜
- 装饰：极简——必要的箭头、边框，禁止漫画/卡通元素
- 反差感来自**内容本身**（数字+故事+对比），不靠视觉花哨

### 6.3 关键交互
| 操作 | 效果 |
|---|---|
| ←/→ | 翻页（同 reveal.js 默认） |
| ↑/↓ | 故事内子页面切换（vertical slides） |
| ESC | 总览所有 slide 缩略图 |
| 1-9 数字键 | 直接跳到对应故事 |
| 0 / Home | 跳回目录页 |
| F | 全屏切换 |
| 点击目录卡片 | 跳到对应故事 |
| 不使用 S | 不开启 speaker notes（按用户要求） |

### 6.4 关键页面布局规范

**封面页**
```
[整屏 logo / 主标题区]
看世界、想问题
—— 美国一些"奇葩"新闻
[副标题] 给四年级的你
```

**目录页（卡片网格 3×3）**
- 9 个故事卡片：缩略图 + 序号 + 标题 + 主题标签 + 时长
- 鼠标悬停高亮 + 点击跳转
- 数字键 1-9 同样跳转

**故事内"对比锚"页布局**
```
┌──────────────────┬──────────────────┐
│  美国             │  中国             │
│ (红色字)          │ (白色字)          │
│                  │                  │
│  16 年            │  39 个月          │
│  没修通           │  通车             │
│                  │                  │
│  $128B           │  $33B            │
│                  │                  │
│ [来源: …]         │ [来源: …]         │
└──────────────────┴──────────────────┘
```

**"想一想"页布局**
- 大字提问
- 下方列出选项 A/B/C（如举手投票）或留白
- 角标标注 ⏱ 30-60 秒

## 7. 素材与数据策略

### 7.1 图片：清单分工
**AI 助手负责**：
- 撰写 `images-manifest.md`，每张图列出：
  - 文件名（如 `01-cahsr-empty-track.jpg`）
  - 用途描述
  - 推荐搜索关键词（中英文）
  - 推荐来源（按优先级：维基百科 CC 图片 → 主流媒体新闻截图 → 其他）
  - **2-3 个候选直链 URL**（提示链接可能失效需核对）
- 优先选择维基百科 / 政府公开数据图（无版权风险）

**用户负责**：
- 按清单访问候选 URL，下载图片
- 重命名为清单约定文件名
- 放进 `images/<story-id>/` 子目录

### 7.2 视频：不使用
- 不嵌入任何视频
- 如有特别想保留的视频片段，可在课件最末加一页"扩展资料"列出 URL，作为课后家长/学生可选观看

### 7.3 数据来源标注
**每条美国 / 中国数据下方必须有小字来源标注**，例如：
```
来源：CHSRA 2024 Project Update Report
```

作用：
- 增强可信度
- 讲师备课时可点击/查询核对
- 对应付家长课堂上的"事实质疑"

### 7.4 数据时效维护
**每讲一次前需快速核对的数据**：
- 加州高铁进度与预算
- ASCE 评级（每 4 年更新）
- 中国高铁里程
- 中国桥梁数

建议在 `stories.json` 里给每条数据加 `last_verified` 字段。

## 8. 内容创作过滤原则

### 8.1 直接不讲（剔除清单）
- ❌ 儿童 / 青少年死亡的具体描述
- ❌ 性 / 毒品 / 酒精滥用
- ❌ 种族冲突的具体场景
- ❌ 阴谋论、未经证实的传闻
- ❌ 强政治立场（特定党派 / 选举议题）
- ❌ 网传"奇葩法律"段子（如"猫做鬼脸违法"等）

### 8.2 淡化处理（提到但不展开）
- 死亡数字可提（"6 人遇难"），但不展示死者照片或尸体
- 消防费故事的"3 狗 1 猫"：用"几只宠物没能逃出来"带过，不渲染
- 校园演练：聚焦"练什么"，不展开具体枪击案
- Key Bridge：聚焦"桥旧+船撞"，不渲染坠桥工人的死亡
- 德州大停电：只讲停电+账单，不讲死亡数字

### 8.3 用语红线
- 全程避免：民主、独裁、资本主义、社会主义、自由、专制、人权（这个词孩子无概念，用"公平""帮助""照顾"代替）
- 不出现：好 / 坏、对 / 错、应该 / 不应该等价值判断词
- 鼓励使用：你觉得 / 想一想 / 为什么 / 后来 / 可能 / 也许

### 8.4 收尾红线
每个故事结尾**必须**给一个"后来怎么样了"的方向：
- 法律改了 / 在改
- 人们在讨论
- 还在修
- 推动了反思

避免任何故事以"问题无解、世界很糟"的氛围收尾。

## 9. 风险与应对

| 风险 | 应对 |
|---|---|
| **学校审查不过** | 上课前 1-2 周把故事池清单发班主任过目，主动征求意见 |
| **家长群反弹**（"为啥讲美国负面"） | 提前准备说辞："这是国际比较视角的公民教育，下次也会讲其他国家"——并真的下一期讲 |
| **现场孩子问"中国有没有这种问题"** | 承认每个国家都有自己的问题，今天只看美国一面，下次可以看世界其他地方。**不否认、不掩饰、不延展** |
| **数据过期** | 每讲一次前快速核对一次：加州高铁进度、ASCE 评级、中国基建数据 |
| **孩子情绪受冲击**（消防费/枪击演练讲完情绪沉重） | 立刻翻到对比锚页+"后来怎么样"，让孩子感受"这事推动了改变"；下一个故事选轻松题材 |
| **故事顺序排错** | 推荐基础顺序：高铁（轻）→ 种菜（轻）→ 接雨水（轻）→ 救护车（中）→ 消防费（中）→ Key Bridge（中）→ 校园演练（重）→ 德州停电（中）→ 基建评级（收尾） |

## 11. 工作分解（高层）

实际实现拆解将在后续 implementation plan 中展开。**开发侧**模块：

1. **内容数据层**：编写 `stories.json`（9 个故事 × 完整字段，含数据来源标注）

**用户侧**任务（不在 implementation plan 范围内）：

7. 与班主任沟通故事池清单（学校审查）
8. 按 `images-manifest.md` 下载图片并放入对应目录
